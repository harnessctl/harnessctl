# Comment Writing Instructions

When adding a comment to an issue to provide an update, use the following structure:

**Status Update:**
Provide a concise summary of the progress made, any key decisions taken, or any failures encountered.

**Artifacts:**

- Link to relevant local files, such as `.specs/`, `.ai.tmp/`, or specific code files modified/created. Do not paste massive text dumps of files; link to them instead.

**Next Steps / Blockers:**

- State clearly what the next immediate steps are.
- Explicitly list any blockers that prevent further progress.

---

**Signature:**
Append the following signature block at the end of your comment to indicate the agent that produced it:

```
opencode-agent: <agent-name>
```
